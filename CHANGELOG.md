### Added

- **Sprint 8 benchmark suite** (S8-T12) — six new BenchmarkDotNet classes under `benchmarks/SmartMapp.Net.Benchmarks/Sprint8/` covering the spec §9.1 performance surface per spec §S8-T12 Outputs list:
  - `FlatMappingBenchmark` — flat 10-property DTO warm-path latency (§9.1 `< 100 ns` target).
  - `NestedMappingBenchmark` — 3-level graph (`Order → Customer → Address`) flattened into a single DTO (§9.1 `< 500 ns` target).
  - `CollectionMappingBenchmark` — 100 / 1 000 / 10 000-item collections, `[Params]`-parameterised (§9.1 `< 100 μs` per 1 K target).
  - `AutoMapperComparisonBenchmark` — SmartMapp.Net vs AutoMapper 13.0.1 over the same flat shape (§S8-T12 AC bullet 2 "vs AutoMapper ratio"). AutoMapper is a benchmark-only reference (`NU1903` warning suppressed — never ships).

### Fixed

- **Sprint 8 holistic review — 4 gaps closed across S8-T01 / S8-T05 / S8-T06 / S8-T08** (+8 regression tests via `SprintEightReviewTests`, 1179 unit + 62 integration = **1241 cumulative green**):
  - **S8-T01 Technical Considerations bullet 4 "Log Info on first Forge"** — the original `ForgedSculptorHost` carried a `// arrives in S8-T05` TODO and never emitted the operator-visibility log the spec calls for. Added an optional `ILogger<ForgedSculptorHost>` ctor dependency plus a `LoggerMessage.Define`-backed one-shot Information event emitted from inside the `LazyThreadSafetyMode.ExecutionAndPublication` forge closure — so it fires exactly once regardless of how many parallel `ISculptor` resolvers race for it. Message shape: *"SmartMapp.Net: forged ISculptor from {AssemblyCount} assemblies; discovered {PairCount} blueprint pair(s) in {DurationMs:F2} ms."* Logger is resolved via `sp.GetService<ILogger<…>>()` so no-logger registrations remain a clean no-op. Host registration switched from pre-constructed-instance (`services.AddSingleton(host)`) to factory form (`services.TryAddSingleton(sp => new ForgedSculptorHost(factory, sp.GetService<ILogger<…>>()))`).
  - **S8-T05 Outputs bullet 3 "default `ValidateOnStartup` is true when `IHostEnvironment.IsDevelopment()`, false otherwise"** — the original `SculptorStartupValidator` always honoured the options flag, so a `Host.CreateDefaultBuilder()` deployment in `Production` still ran validation. Introduced `SculptorOptions.IsValidateOnStartupExplicitlySet` (flips `true` on any setter call, even to the default-matching value) plus an optional `IHostEnvironment` ctor dependency on the validator. New `ShouldValidate()` helper applies three-level precedence: (1) explicit setter wins; (2) no `IHostEnvironment` → fall through to the options flag (builder-only consumers keep fail-fast); (3) otherwise validate only in `Development`. New Debug-level "validation skipped for host environment …" log event preserves operator traceability. Validator DI registration switched to factory form (`services.AddSingleton<IHostedService>(sp => new SculptorStartupValidator(… sp.GetService<IHostEnvironment>()))`) so the optional dep works without MSDI complaining about unresolved ctor params. Sample `MinimalApi`'s `Program.cs` now sets `options.ValidateOnStartup = true` explicitly so its validator demo fires regardless of launch profile; pre-existing unit/integration tests that asserted on always-on validation updated to either pin `Development` or opt in explicitly.
  - **S8-T06 Outputs bullet 3 "`src/SmartMapp.Net/Runtime/ProjectionExpressionCache.cs`"** — projection cache lived inline on `ForgedSculptorConfiguration.ProjectionCache` as a raw `ConcurrentDictionary<TypePair, LambdaExpression>`. Extracted a dedicated `Runtime/ProjectionExpressionCache.cs` class exposing just `TryGet`, `GetOrAdd(TypePair, Func<TypePair, TArg, LambdaExpression>, TArg)`, and `Count` — keeps the projection memoisation semantics isolated from the rest of the configuration surface without leaking the full `ConcurrentDictionary` API (consumers must not mutate post-forge). `SculptorProjectionBuilder.BuildOrGet` continues to use the same call shape; no behaviour change, only file-structure alignment.
  - **S8-T08 Outputs bullet 2 "`src/SmartMapp.Net/Composition/CompositionRuleBuilder.cs`"** — composition accumulator lived as `Abstractions/CompositionRule<TTarget>`. Moved the file to `Composition/CompositionRuleBuilder.cs`, renamed the class to `CompositionRuleBuilder<TTarget>`, and updated the sole production call site (`BlueprintBuilder.Compose<TTarget>`) + test reference (`CompositionBlueprintTests`). Class is internal so the rename is non-breaking; cref references on `ICompositionRuleInternal` and `CompositionBlueprint` adjusted to point at the new type.
- **`ForgeBenchmark.Forge_100Pairs` recorded `NA` in the original baseline** (S8-T12 review) — the first attempt rebound `ForgeSourceA/B/C → ForgeTargetA/B/C` 100 times each, hitting `BlueprintBuilder`'s pair-uniqueness invariant and surfacing as `System.InvalidOperationException: Duplicate binding`. Fixed by extracting `HundredPairsBlueprint` with 100 unique closed generics; refreshed baseline now records 31.74 ms / 2.08 MB for the 100-pair forge.
- **`consolidate-baseline.ps1` crashed on Windows PowerShell 5.1** (S8-T12) — the null-conditional `?.` / `??` operators in the path-resolve call aren't supported on the pinned PowerShell runtime; replaced with an explicit `IsPathRooted` check before `Join-Path`.
- **Sprint 8 S8-T11 integration tests had three gaps** (S8-T11 review) — missing shared `AppHostFixture` (closed with `[CollectionDefinition]` + `SharedHostMapperTests` / `SharedHostSelectAsTests` proving single-forge + single-seed semantics), SQL-shape assertion was an InMemory placeholder (closed via SQLite-backed `SelectAsSqliteSqlShapeTests` asserting `SELECT ... AS "CustomerFirstName" ... INNER JOIN "Customers"`), and warnings-only startup scenarios were absent (closed via `WarningsBlueprint` + two new `StartupValidationTests`). +9 regression tests.
- **`ConsoleSampleSmokeTests` was locale-sensitive** (S8-T09 review) — scenarios printed `Balance` / `Tax` / `Total` with the current culture's decimal separator so a CI runner on a non-en-US locale would fail the stdout-substring assertions. Fixed by switching every formatter to `CultureInfo.InvariantCulture`.
- **Top-level README Quick Start was too long** (S8-T10 review) — spec §S8-T10 AC bullet 6 caps the snippet at 15 lines; the original weighed in at 37. Consolidated to a single 12-line shell + csharp block.

### Spec Deviations

- **Benchmarks target `net10.0` only** — spec §S8-T12 Constraints bullet 3 calls for `net8.0;net9.0`. `BenchmarkDotNet` 0.14's `RuntimeMoniker` enum caps at `Net90`, and `global.json` pins the `net10.0` SDK. The dev SDK catalogue does not currently ship `net8.0` / `net9.0` runtimes, so attaching `[SimpleJob(RuntimeMoniker.Net80)]` / `Net90` would fail benchmark runs on the dev box. The benchmark class shapes are stable; Sprint 9 attaches the moniker-pinned jobs once the CI runner provisions those runtimes. Retro tracks this as carry-over.
- **Mutation-testing floor at ≥ 75 %** — spec §S8-T12 AC bullet 5 targets ≥ 80 % with ≥ 75 % as the explicit RC-acceptable stretch. CI job runs with `--threshold-break 75`; hardened to 80 once the first CI run records a baseline.
- **`dotnet package validate` CLI verb** — spec §S8-T12 Unit-Tests bullet 2 calls for a literal shell-out. The verb does not exist in the current .NET SDK; the supported validation surface is the `EnablePackageValidation` MSBuild property that runs inline at pack time. Implemented via the MSBuild property with `Pack_InvokesPackageValidator_AndReportsEnabled` test asserting the verbose-log marker.

### Added — Sprint 2: Convention Engine

- **`IPropertyConvention`** interface — contract for property-level conventions with `Priority` and `TryLink`.
- **`PropertyAccessProvider`** — compiled `IValueProvider` for reading a single property/field from origin via `Expression.MakeMemberAccess`.
- **`ExactNameConvention`** (Priority 100) — case-insensitive exact name matching; prefers exact-case and properties over fields.
- **`CaseConvention`** (Priority 200) — cross-case matching (snake_case, camelCase, PascalCase, SCREAMING_SNAKE, kebab-case) via `NameNormalizer`.
- **`NameNormalizer`** — static utility to segment, normalize, and compare member names across casing conventions. Includes `ReadOnlySpan<char>` optimization on .NET 8+.
- **`PrefixDroppingConvention`** (Priority 250) — strips configurable prefixes (`Get`, `m_`, `_`, `Str`) and suffixes (`Field`, `Property`, `Prop`) before matching.
- **`MethodToPropertyConvention`** (Priority 275) — links target properties to parameterless origin methods (e.g., `GetFullName()` → `FullName`).
- **`MethodAccessProvider`** — compiled `IValueProvider` for invoking a parameterless method via `Expression.Call`; null-safe origin handling.
- **`FlatteningConvention`** (Priority 300) — decomposes flattened target names into origin member chains (e.g., `CustomerAddressCity` → `Customer.Address.City`) using recursive greedy prefix matching with backtracking. Max depth 5.
- **`ChainedPropertyAccessProvider`** — `IValueProvider` navigating a `MemberInfo[]` chain with null-safe traversal via composed compiled accessors.
- **`UnflatteningConvention`** (Priority 350) — reverse of flattening; maps flat origin members to nested target structures with intermediate object creation via `Activator.CreateInstance`.
- **`UnflatteningValueProvider`** — reads multiple flat origin members and assembles them into a nested target object.
- **`AbbreviationConvention`** (Priority 400) — bidirectional abbreviation expansion via configurable alias dictionary (default: `Addr`↔`Address`, `Qty`↔`Quantity`, `Amt`↔`Amount`, `Desc`↔`Description`, `Num`↔`Number`, and more). Uses `NameNormalizer` for multi-segment expansion.
- **`StructuralSimilarityScorer`** — computes similarity score (0.0–1.0) between two types based on member name and type compatibility. Returns detailed `StructuralSimilarityResult` with matched/unmatched member lists.
- **`ConventionPipeline`** — orchestrator that runs conventions in priority order to build `List<PropertyLink>` for any origin/target pair. Includes `StrictMode` (throws on unlinked required members), `IgnoreUnlinked`, and `CreateDefault()` factory.
- **`NullValueProvider`** — internal no-op provider for skipped (unlinked) property links.
- **`ITypeConvention`** interface — contract for type-level conventions that discover which type pairs should be mapped.
- **`NameSuffixTypeConvention`** — auto-pairs types by matching base names with configurable suffixes (9 defaults: `Dto`, `ViewModel`, `Vm`, `Model`, `Response`, `Request`, `Command`, `Entity→Dto`, `Entity→ViewModel`). Verifies structural similarity above threshold (default 0.7).
- **`ConventionMatch`** factory methods — `CaseNormalized`, `Unflattened`, `PrefixDropped`, `MethodToProperty`, `Abbreviation`, `None`.
